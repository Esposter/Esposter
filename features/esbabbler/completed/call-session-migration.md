# Esbabbler — Call Session Migration (Phase 1b)

Decoupled calls from room-scoped identity. Enables shareable meeting links (like Google Meet / Microsoft Teams).

---

## What Changed

### New: `callSessionsInMessage` table

Persistent 1:1 anchor per room. Session survives server restarts; participants do not.

```ts
callSessionsInMessage = pgTable(
  "call_sessions",
  {
    id: uuid().primaryKey().defaultRandom(), // callSessionId — key for all call ops
    roomId: uuid()
      .notNull()
      .unique()
      .references(() => roomsInMessage.id, { onDelete: "cascade" }),
    token: text().notNull().unique(), // 12-char alphanumeric, shareable
  },
  { schema: messageSchema },
);
```

### Renamed: `invitesInMessage.code` → `.token`

`CODE_LENGTH` → `INVITE_TOKEN_LENGTH = 8`. Both invite and call tokens generated by `createToken(length)`.

### New: `createToken(length)` — cryptographically secure

Replaces `createCode` (was `Math.random`). Uses `crypto.getRandomValues` with a 62-char alphanumeric set.

```ts
export const createToken = (length: number): string => {
  const bytes = crypto.getRandomValues(new Uint8Array(length));
  return Array.from(bytes, (byte) => TOKEN_CHARACTERS.charAt(byte % TOKEN_CHARACTERS.length)).join("");
};
```

### Procedure changes (`server/trpc/routers/room/call.ts`)

| Procedure         | Auth             | Input                        | Notes                                                  |
| ----------------- | ---------------- | ---------------------------- | ------------------------------------------------------ |
| `readCallSession` | member           | `{ roomId }`                 | New. Upserts session row via `getOrCreateCallSession`  |
| `joinCall`        | member           | `{ roomId }`                 | Returns `{ callSessionId, participants }` (not roomId) |
| `joinCallByToken` | authed (no room) | `{ token }`                  | New. Finds session by token; no room membership needed |
| `leaveCall`       | authed (no room) | `{ callSessionId }`          | Was `{ roomId }`                                       |
| `setMute`         | authed (no room) | `{ callSessionId, isMuted }` | Was `{ roomId, isMuted }`                              |
| `sendSignal`      | authed (no room) | `{ callSessionId, payload }` | Was `{ roomId, payload }`                              |
| `onJoinCall` etc. | authed (no room) | `callSessionId` (UUID)       | All subscriptions take callSessionId, not roomId       |

`requireCallSession(db, callSessionId)` — verifies session exists before allowing subscription.

### `getOrCreateCallSession` — race-safe upsert

```ts
// 1. findFirst by roomId → return if exists
// 2. Insert loop (3 tries) with fresh token each attempt (handles unique constraint collision)
// 3. Fallback findFirst after all inserts fail (concurrent request won the race)
// 4. Throw UNPROCESSABLE_CONTENT if still not found
```

### Client store (`app/store/message/room/call.ts`)

| Ref                          | Purpose                                                         |
| ---------------------------- | --------------------------------------------------------------- |
| `activeCallSessionId`        | Session the user is IN — drives `leaveCall`, `setMute`, signals |
| `currentRoomCallSessionId`   | Session for the VIEWED room — set by `useCallSubscribables`     |
| `callRoomId`                 | Room ID of active call — only for admin action room checks      |
| `callSessionParticipantsMap` | `Map<callSessionId, CallParticipant[]>` (was `Map<roomId, …>`)  |

### `useCallSubscribables` flow

1. Room entry → `readCallSession({ roomId })` → `setCurrentRoomCallSessionId(callSessionId)`
2. `readCallParticipants({ callSessionId })` → `setParticipants`
3. If already in call: re-join via `joinCall()` (uses `currentRoomCallSessionId` already set)
4. Subscribe `onJoinCall / onLeaveCall / onSetMute` with `callSessionId`
5. Room exit → `leaveCall.mutate({ callSessionId })`, `setCurrentRoomCallSessionId("")`, `cleanupAll()`

### Shareable call link

`/call/[token]` page (frontend not yet implemented) calls `joinCallByToken({ token })`.
Any authenticated user with the token can join regardless of room membership.

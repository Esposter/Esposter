# Esbabbler — v5 (Completed)

All items delivered. Remaining quick-wins and DM management moved to v6.

---

## Call — Phased Upgrade (audio → video → screenshare)

### Phase 1 — Rename identifiers to `call` ✅

Source identifiers, file paths, tRPC keys, store IDs, and spec docs renamed to `call*`. Enum values updated: `MessageType.Call` and `AdminActionType.KickFromCall`.

---

### Phase 1b — Persistent call sessions + shareable links ✅

`callSessionsInMessage` (Postgres) as stable 1:1 anchor per room and roomless calls. Call session IDs are 12-char public IDs (`CALL_ID_LENGTH`). All in-memory maps and event emitter events key by `callSessionId`. Added `readCallSessionId`, `readCallSession`, `createCall`, `joinCall`. Renamed `invitesInMessage.code` → `.token` and `createCode` → `createToken` (now `crypto.getRandomValues`). Client store tracks `activeCallSessionId`, `currentRoomCallSessionId`, `callRoomId`. See `completed/call-session-migration.md`.

---

### Phase 2 — LiveKit migration (audio parity) ✅

- LiveKit Cloud free tier provisioned; `LIVEKIT_URL`, `LIVEKIT_API_KEY`, `LIVEKIT_API_SECRET` in `.env.example`
- `server/api/webhooks/livekit.post.ts` — webhook receiver for participant joined/left events
- `livekit-server-sdk`, `livekit-client`, `@livekit/track-processors` added to catalog
- `joinCall` / `joinCallByRoomId` generate LiveKit JWT, return `{ livekitUrl, livekitToken }`
- Deleted `sendSignal` + `onSendSignal`; added `onVideoChanged` subscription
- `store/message/room/liveKit.ts` — all RTCPeerConnection / ICE logic replaced with LiveKit `Room`
- `store/message/room/call/index.ts`, `media.ts`, `participant.ts` refactored

---

### Phase 3 — Video (camera tracks) ✅

`CallParticipant.isCameraEnabled`, camera toggle button, device selector, virtual backgrounds via `@livekit/track-processors`, `Content/Call/Participant/Tile.vue` (camera + avatar fallback + speaking ring + mute badge), adaptive video grid in `Content/Call/View.vue`.

---

### Phase 4 — Screenshare ✅

`setScreenShare` in liveKit store, `Content/Call/ScreenShare/Button.vue`, `Content/Call/ScreenShare/Stage.vue` (presenter view), `Content/Call/InviteCard.vue` (call link sharing), `Content/Call/JoinNotice/Index.vue`. `AdminActionType.StopScreenShare` — server-side LiveKit enforcement revokes screen-share publish sources; requires `MuteMembers` permission.

---

### Phase 5 — Call Lobby / Waiting Room (Knock & Admit) ✅

Standalone calls only. States on `pages/calls/[id].vue`: `idle` (pre-join preview), `knocking` ("Waiting to be admitted..."), `joined` (full `CallView`). Creator goes directly to `joined`; all others must knock.

Server procedures: `knockCall`, `admitKnocker`, `dismissKnocker`, `onKnockCall`, `onKnockerAdmitted`, `onKnockerDismissed`. In-memory `callKnockerMap` and `callAdmittedParticipantMap`. Client store: `store/message/room/call/knocker.ts`. Components: `Content/Call/PreJoin/Index.vue`, `Content/Call/Waiting.vue`, `Content/Call/JoinNotice/KnockerItem.vue`.

---

## Per-room nicknames ✅

`nickname text NOT NULL DEFAULT ""` (max 32 chars) on `usersToRooms`. `updateUserToRoom({ roomId, nickname })` — own nick: any member; others' nick: `ManageNicknames` permission (implemented in `server/services/message/updateUserToRoom.ts`). `nicknameMap` in store. Falls back to global username when empty.

---

## Unread indicators ✅

Unread room dot: compares `usersToRooms.lastMessageAt` to `rooms.updatedAt`; `lastMessageAt` updated on room focus/navigation switch; filled dot in sidebar when behind and room is not active. No new column — reuses existing fields.

---

## Role mentions ✅

`@RoleName` in composer suggestion list via second `MentionType.Role` source on the TipTap Mention extension. Push notifications: `getRoleMentionCondition.ts` detects role mention spans and queries `usersToRoomRoles` for members of that role, respecting `notificationType`. Role color applied client-side from `memberMap` store.

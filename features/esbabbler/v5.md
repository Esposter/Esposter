Spec: [`specs/call.md`](specs/call.md) | [`specs/screenshare.md`](specs/screenshare.md)

### Phase 1 ‚Äî Rename identifiers to `call` ‚úÖ (pure refactor, no behavior change)

Source identifiers, file paths, tRPC keys, store IDs, and spec docs renamed to `call*`.
Enum values also updated: `MessageType.Call` and `AdminActionType.KickFromCall`. Existing stored values in Azure Table Storage using the old strings are stale.

---

### Phase 1b ‚Äî Persistent call sessions + shareable links ‚úÖ

Introduced `callSessionsInMessage` (Postgres) as a stable 1:1 anchor per room. All in-memory maps and event emitter events now key by `callSessionId` (UUID) instead of `roomId`. Added `readCallSession({ roomId })` (upserts on room entry) and `joinCallByToken({ token })` (auth-only, no room membership) for shareable `/call/[token]` meeting links. Renamed `invitesInMessage.code` ‚Üí `.token` and `createCode` ‚Üí `createToken` (now uses `crypto.getRandomValues`). Client store tracks `activeCallSessionId` (drives tRPC ops), `currentRoomCallSessionId` (viewed room), and `callRoomId` (admin action room checks). See `completed/call-session-migration.md` for full details.

## Comfort gap analysis

After v4 the platform has: messaging, reactions, threads, drafts, calls, DMs, friends, full RBAC, full moderation (timeout/ban/warn/softban/word filter), slash commands, slowmode, read-only channels, system messages, link previews, file uploads, presence, user profiles, global search, push notifications, inbound webhooks, categories, pinned messages.

Core remaining friction points vs Discord: no role mentions, no room identity (icon), no per-room nickname, no group DM management. Resolved: unread indicators. Dropped: message permalinks (not needed).

## Room Identity

- [ ] **Room avatar / icon** ‚Äî `image: text("image").notNull().default("")` on `rooms` (column exists but is currently nullable ‚Äî migration needed to add NOT NULL + DEFAULT ''); upload via existing `AzureContainer.PublicUserAssets` SAS flow (reuse `uploadProfileImage` pattern from user profile); displayed in the sidebar room list, room header, and member hover cards; editable in room settings behind `ManageRoom`; empty string = no image set

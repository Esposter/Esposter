### Phase 1 — Rename identifiers to `call` ✅ (pure refactor, no behavior change)

Source identifiers, file paths, tRPC keys, store IDs, and spec docs renamed to `call*`.
Enum values also updated: `MessageType.Call` and `AdminActionType.KickFromCall`. Existing stored values in Azure Table Storage using the old strings are stale.

---

### Phase 1b — Persistent call sessions + shareable links ✅

Introduced `callSessionsInMessage` (Postgres) as a stable 1:1 anchor per room and roomless calls. Call session IDs are 12-character public IDs (`CALL_ID_LENGTH`), not UUIDs. All in-memory maps and event emitter events now key by `callSessionId` instead of `roomId`. Added `readCallSessionId({ roomId })` for room observers, `readCallSession({ id })` for standalone call validation, `createCall()` for roomless sessions, and `joinCall({ id })` for shareable `/calls/[id]` meeting links. Renamed `invitesInMessage.code` -> `.token` and `createCode` -> `createToken` (now uses `crypto.getRandomValues`). Client store tracks `activeCallSessionId` (drives tRPC ops), `currentRoomCallSessionId` (viewed room), and `callRoomId` (admin action room checks). See `completed/call-session-migration.md` for full details.

### Phase 5 — Call Lobby / Waiting Room (Knock & Admit) ✅

Scope: standalone calls only (`/calls/[id]`). Room calls are already gated by room membership + RBAC.

**Problem**: Any authenticated user with the share link can join a standalone call immediately. No way for the call creator to admit or reject participants.

**Design (no separate page — overlay in `/calls/[id]`):**

States on `pages/calls/[id].vue`:

| State      | What user sees                                                 |
| ---------- | -------------------------------------------------------------- |
| `idle`     | Pre-join screen (camera/mic preview, "Request to Join" button) |
| `knocking` | "Waiting to be admitted…" overlay; cancel button               |
| `joined`   | Full `CallView`                                                |

The call creator is stored as `call_sessions.userId` and transitions directly to `joined` on `/calls/[id]`. Everyone else goes through `idle` -> `knocking` -> `joined`; the server rejects direct `joinCall({ id })` unless the caller is the creator or has been admitted.

**Server procedures** (all in `server/trpc/routers/room/call.ts`):

| Procedure            | Auth            | Input                          | Purpose                                                                                                                |
| -------------------- | --------------- | ------------------------------ | ---------------------------------------------------------------------------------------------------------------------- |
| `knockCall`          | authed          | `{ id }`                       | Add caller to `callKnockerMap`; emit `onKnockCall`; return nothing                                                     |
| `admitKnocker`       | creator in call | `{ callSessionId, sessionId }` | Remove from `callKnockerMap`; add one-time admission; emit `onKnockerAdmitted` (to knocker); they then call `joinCall` |
| `dismissKnocker`     | creator in call | `{ callSessionId, sessionId }` | Remove from `callKnockerMap`; emit `onKnockerDismissed`                                                                |
| `onKnockCall`        | creator in call | `callSessionId`                | Subscription; fires when someone knocks                                                                                |
| `onKnockerAdmitted`  | authed          | `callSessionId`                | Subscription; fires on the knocker when admitted                                                                       |
| `onKnockerDismissed` | authed          | `callSessionId`                | Subscription; fires on the knocker when dismissed                                                                      |

`callKnockerMap` — `Map<callSessionId, Map<sessionId, CallParticipant>>` in-memory, same pattern as `callSessionParticipantMap`.
`callAdmittedParticipantMap` — `Map<callSessionId, Set<sessionId>>`; `admitKnocker` adds the session ID, and `joinCall` consumes it so admitted access is one-time.

**Permissions**: admission control is host-only for standalone calls. The caller must have an entry in `callSessionParticipantMap` and must match `callSessionsInMessage.userId`. No RBAC is involved because standalone calls have no room membership.

**Client stores**:

- `store/message/room/call/knocker.ts` — owns `knockingCallSessionId`, `joinCallOptions`, `knockers`, and `knockCall` / `admitKnocker` / `dismissKnocker`
- `store/message/room/call/index.ts` — resets knocker state on leave and still owns active call membership

**Components:**

- `pages/calls/[id].vue` — render `CallPreJoin` when `!activeCallSessionId && !knockingCallSessionId`; render `CallWaiting` when `knockingCallSessionId`; render `CallView` when `activeCallSessionId`
- `Content/Call/PreJoin/Index.vue` — camera/mic preview, "Request to Join" button (calls `knockCall`)
- `Content/Call/Waiting.vue` — "Waiting to be admitted..." + cancel button (clears `knockingCallSessionId`; dismissal redirects to `/calls`)
- `Content/Call/JoinNotice/Index.vue` — shows knocker list with admit/dismiss actions before the standard join notice
- `Content/Call/JoinNotice/KnockerItem.vue` — per-knocker "Let in" / "Dismiss" controls (calls `admitKnocker` / `dismissKnocker`)

**Flow:**

```text
Knocker                        Server                    Participants
  |-- knockCall({ id }) ------->|                              |
  |                             |-- onKnockCall emit -------->|
  |-- subscribe onKnockerAdmitted/Dismissed                   |
  |   (show "Waiting to be admitted...")                      |
  |                             |<-- admitKnocker ------------|
  |                             |-- onKnockerAdmitted emit -->|
  |<-- onKnockerAdmitted -------|                              |
  |-- joinCall({ id }) -------->|                              |
  |<-- { callSessionId, ... }---|                              |
  |-- room.connect() ------------------------------------------
```

**Bug fix also in this phase**: `InviteCard.vue` no longer shown inside the room call dialog (`Panel/Dialog.vue`). It now only appears on standalone call pages (`!callRoomId`). Already fixed.

**Bug fix also from spec review**: `useCallIdSubscribables` now clears `knockingCallSessionId` on unmount when the user navigates away while waiting, so stale waiting-room state cannot carry into the next call page.

## Comfort gap analysis

After v4 the platform has: messaging, reactions, threads, drafts, calls, DMs, friends, full RBAC, full moderation (timeout/ban/warn/softban/word filter), slash commands, slowmode, read-only channels, system messages, link previews, file uploads, presence, user profiles, global search, push notifications, inbound webhooks, categories, pinned messages.

Core remaining friction points vs Discord: no role mentions, no room identity (icon), no per-room nickname, no group DM management. Resolved: unread indicators. Dropped: message permalinks (not needed).

## Call — Phased Upgrade (audio → video → screenshare)

Spec: [`specs/call.md`](specs/call.md) | [`specs/screenshare.md`](specs/screenshare.md)

---

### Phase 1 — Rename identifiers to `call` ✅ (pure refactor, no behavior change)

Source identifiers, file paths, tRPC keys, store IDs, and spec docs renamed to `call*`.
Enum values also updated: `MessageType.Call` and `AdminActionType.KickFromCall`. Existing stored values in Azure Table Storage using the old strings are stale.

---

### Phase 1b — Persistent call sessions + shareable links ✅

Introduced `callSessionsInMessage` (Postgres) as a stable 1:1 anchor per room. All in-memory maps and event emitter events now key by `callSessionId` (UUID) instead of `roomId`. Added `readCallSession({ roomId })` (upserts on room entry) and `joinCallByToken({ token })` (auth-only, no room membership) for shareable `/call/[token]` meeting links. Renamed `invitesInMessage.code` → `.token` and `createCode` → `createToken` (now uses `crypto.getRandomValues`). Client store tracks `activeCallSessionId` (drives tRPC ops), `currentRoomCallSessionId` (viewed room), and `callRoomId` (admin action room checks). See `completed/call-session-migration.md` for full details.

## Comfort gap analysis

After v4 the platform has: messaging, reactions, threads, drafts, calls, DMs, friends, full RBAC, full moderation (timeout/ban/warn/softban/word filter), slash commands, slowmode, read-only channels, system messages, link previews, file uploads, presence, user profiles, global search, push notifications, inbound webhooks, categories, pinned messages.

Core remaining friction points vs Discord: no role mentions, no room identity (icon), no per-room nickname, no group DM management. Resolved: unread indicators. Dropped: message permalinks (not needed).

## Quick Wins

1. **Collapsible embeds** — client-side localStorage, zero server cost
2. **Room avatar / icon** — Azure Blob SAS reusing existing upload flow
3. **Per-room nickname** — one nullable column on `usersToRooms`

## Room Identity

- [ ] **Room avatar / icon** — `image: text("image").notNull().default("")` on `rooms` (column exists but is currently nullable — migration needed to add NOT NULL + DEFAULT ''); upload via existing `AzureContainer.PublicUserAssets` SAS flow (reuse `uploadProfileImage` pattern from user profile); displayed in the sidebar room list, room header, and member hover cards; editable in room settings behind `ManageRoom`; empty string = no image set
- [x] **Per-room nickname** — `nickname text NOT NULL DEFAULT ""` (max 32 chars) on `usersToRooms`; `updateUserToRoom({ roomId, nickname })` mutation (own nick: any member; others' nick: `ManageNicknames` permission — pending); `readNicknames({ roomId, userIds })` returns public-only fields for members; `readMyUsersToRooms({ roomIds })` returns full private row for self; `nicknameMap` in store separate from private self-data; **My Profile** room settings tab for all members; falls back to global username when empty — [ ] `ManageNicknames` permission for setting others' nicknames still pending

---

## Unread Indicators

- [x] **Unread room dot** — compare `usersToRooms.lastMessageAt` (already written by slowmode on each send) to `rooms.updatedAt` to determine "has new messages since last visit"; update `lastMessageAt` on room focus/navigation switch (currently only updated on message send); show a filled dot in the sidebar room row when `lastMessageAt < rooms.updatedAt` and room is not currently active; no new column needed — reuses existing fields; no per-message reads; no unread count (Discord shows counts, we skip to avoid read-receipt semantics already deferred twice)

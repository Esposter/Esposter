# Esbabbler — v4 Roadmap

All features are free-to-implement (no paid infra) unless explicitly noted.
Items leverage existing Azure Table Storage, Postgres, and Azure Function infrastructure.
Strikethroughs are acknowledged but deferred with rationale.

---

## Moderation Extensions

Building on v3's `moderationRouter`, `AdminActionType`, and `ModerationLog` Azure Table — no new infrastructure.

- [ ] **Warn action** — add `AdminActionType.Warn` with an optional `reason: string` field; write to `ModerationLog` like other actions; `onAdminAction` subscription delivers a snackbar to the target; warn count visible in audit log
- [ ] **Softban** — `AdminActionType.SoftBan`: kick + mark the user's last N (default 100) Azure Table messages as `deletedAt = now()`; behind `BanMembers` permission; useful for clearing spam without a permanent ban
- [ ] **Basic word filter** — per-room blocked-words list stored as `string[]` on a new `roomFilters` Postgres table; `createMessage` checks before persisting and throws `FORBIDDEN` with a user-visible reason; moderators with `ManageMessages` bypass; case-insensitive substring match
- [ ] **Auto-prune timed-out messages** — when a timeout expires (`timeoutUntil < now()`) clear `timeoutUntil` via an Azure Function daily timer scan; prevents stale timeouts from requiring a manual unset

## Room Management

- [ ] **Read-only / announcement channels** — add `isReadOnly: boolean` (default `false`) to `rooms` Drizzle schema + migration; `createMessage` rejects with `FORBIDDEN` if `isReadOnly && !hasPermission(ManageMessages)`; editable in room settings behind `ManageRoom`; shown with a megaphone icon in the sidebar
- [ ] **System messages for join / leave** — post `MessageType.JoinRoom` / `MessageType.LeaveRoom` when `usersToRooms` rows are inserted / deleted; uses existing `createMessage` + `messageEventEmitter` path; no new storage; rendered as a compact system row (same style as `PinMessage`)
- [ ] **Slowmode** — add `slowmodeSeconds: integer | null` to `rooms` + migration; add `lastMessageAt: timestamp | null` to `usersToRooms` + migration; `createMessage` reads both, rejects with `TOO_MANY_REQUESTS` if `now() - lastMessageAt < slowmodeSeconds`; update `lastMessageAt` on successful send; users with `ManageMessages` bypass; configurable in room settings

## Message Improvements

- [ ] **Message drafts** — persist draft per room in `localStorage` keyed by `roomId`; restore on room switch; clear on successful send; pure client-side, zero server cost
- [ ] **Thread view** — add a "View Thread" action on any message that has replies (`replyRowKey` chain); opens a right-side panel (`ThreadPanel.vue`) showing the root message + all messages where `replyRowKey = rootRowKey`; server: new `readThread({ rootRowKey, roomId })` query filtering Azure Table by `replyRowKey`; no new storage

## Slash Commands

Building on the existing slash-command infrastructure in `shared/models/message/slashCommands/`.

- [ ] **`/remind [duration] [text]`** — client schedules a `setTimeout` (tab must stay open); on fire, calls `createMessage` with the reminder text prefixed by "⏰ Reminder:"; purely client-side for short durations (≤ 1 h); no server or Queue needed for the casual use case
- [ ] **`/topic [text]`** — sets the room topic (a new `topic: string | null` column on `rooms`); `updateRoom` tRPC mutation behind `ManageRoom`; displayed in a subtitle below the room name in the header; behind `ManageRoom` permission
- ~~**`/remind` with Azure Queue** — reliable cross-tab delivery via Azure Storage Queue + Azure Function — (Deferred: extra infra cost for a feature most users can get from their OS.)~~

## Performance / Infrastructure

- [ ] **`usersToRooms` index on `timeoutUntil`** — `assertNotTimedOut` queries `usersToRooms` on every `createMessage`; add a partial index `WHERE timeoutUntil IS NOT NULL` to make the common (non-timed-out) fast-path nearly free
- [ ] **Azure Table batch reads in `readMessagesByRowKeys`** — currently issues one `getEntity` per rowKey; use `listEntities` with an OData `rowKey eq X or rowKey eq Y …` filter to collapse into a single round-trip for the common case (≤ 15 rowKeys)

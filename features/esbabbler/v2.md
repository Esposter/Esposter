# Esbabbler — v2 Feature Roadmap

All features below are free-to-implement (no paid infra) unless explicitly noted.
Items are grouped by theme; strikethroughs are acknowledged but deferred with rationale.

---

## Completing v1 Open Items

- [ ] **Blocked users** — implement the reserved `FriendshipStatus.Blocked` variant; block hides user from `searchUsers` in both directions; `sendFriendRequest` rejects if a block row exists in either direction
- [ ] **Friend request notifications** — Web Push on `sendFriendRequest` using the existing Event Grid + Web Push pipeline; opt-in (default on, per-user toggle)
- ~~**DM unread badges**~~ _(Deferred — read receipts / lastSeenAt tracking is not good UX for this platform.)_
- [x] **Voice UI refactor + DM voice** — move voice from the sidebar to a header button + active-call panel above the message list; add mobile header overflow menu; enable DM voice; add VoiceCall system message; see [`voice-refactor.md`](voice-refactor.md)

---

## Room Management

- [ ] **Room categories** — a `categories` table (`id`, `guildId`, `name`, `position`); rooms gain a nullable `categoryId` FK; sidebar renders collapsible category groups with drag-to-reorder; mirrors Discord's category model but simpler (no per-category permission overrides)
- [ ] **Room topic / description** — a `topic` text column on `rooms`; shown beneath the room name in the header bar; editable by room admins via room settings
- [x] **Pinned messages list** — `PinnedMessagesMenuButton.vue` in the room header; cursor-paginated list via `useReadPinnedMessages`, renders via `MessageModelMessageSearchList`
- [x] **Edit own messages** — tRPC `updateMessage` mutation restricted to `authorId = session.id`; edited messages show an "(edited)" label; ~~no edit history~~ (still deferred — store only the latest text, not a version log)
- [x] **Delete own messages** — tRPC `deleteMessage` mutation restricted to `authorId = session.id` (admins can also delete any); renders as a tombstone or removes the row entirely

---

## Slash Commands

- [x] **`/me [action]`** — parameterized; posts `*action text*` as markdown italic via regular `MessageType.Message` (no separate MessageType needed)
- [x] **`/flip`** — immediate; client-side coin flip; posts "🪙 Heads" or "🪙 Tails"
- [x] **`/shrug`** — parameterized; optional text prefix + `¯\_(ツ)_/¯`
- [x] **`/tableflip`** — immediate; posts `(╯°□°）╯︵ ┻━┻`
- [x] **`/unflip`** — immediate; posts `┬─┬ノ( º _ ºノ)`

All are pure client-side `createMessage` calls — zero server logic beyond the existing `createMessage` path. Commands with parameters use inline param input mode in the composer (Discord-style) with required field validation.

## Mentions & Notifications

- [ ] **@here** — mention all currently-online room members; resolved server-side in `createMessage` by querying `userStatuses` for `status = Online`; excluded from `@everyone` if you only want active users
- [ ] **@everyone** — mention all room members; resolved from `usersToRooms`; restricted to room admins/owners to prevent abuse
- ~~**Per-room notification preference**~~ _(Already implemented — `NotificationType` enum (`All` / `DirectMessage` / `Never`) on `usersToRooms`; enforced in `getPushSubscriptionsForMessage` in `packages/db`.)_

## Presence & Profiles

- [ ] **Custom status message** — a short free-text `statusMessage` (max 64 chars) alongside the existing `Online/Idle/DND/Offline` selector; stored on `userStatuses`; shown in member hover cards and the presence panel
- [ ] **User profile card** — clicking a username/avatar anywhere opens a hover card (or a modal on mobile) with: avatar, display name, username, custom status, mutual rooms, and a "Send Friend Request" / "Send Message" CTA depending on friendship state
- ~~**"Currently Playing" activity status**~~ _(Requires a desktop app or browser extension to detect active applications; not feasible in a pure web platform.)_

---

## Search Improvements

- [ ] **Filtered message search** — extend the existing Ctrl+K command palette with filter chips: `from:@user`, `in:#room`, `before:/after: [date]`; server-side, compose `WHERE` clauses on `authorId`, `roomId`, and `createdAt`; keep the existing full-text `ILIKE` for the text term

---

## Voice Channel Improvements

- [ ] **Deafen toggle** — mute all incoming audio without disconnecting; client-side only (`AudioContext` gain node set to 0); broadcasts `isDeafened` alongside `isMuted` via `onMuteChanged`; shown as a headphone-slash badge on the participant avatar
- [ ] **Moderator force-mute** — room admins can force-mute a participant; `setMute` gains an optional `targetUserId` param; `onMuteChanged` carries a `forcedByAdminId` field; targeted client sets `isMuted = true` and disables the local toggle until un-muted by the admin
- ~~**Video / screen share**~~ _(Requires an SFU for more than 2 viewers; deferred until LiveKit is provisioned — see voice-channel.md scale path.)_

## UI / UX Refactors

- [ ] **Collapsible sidebar sections** — the Left Sidebar "Rooms", "Direct Messages", and "Friends" sections remember collapsed state in `localStorage`; each header has a chevron toggle; reduces visual noise when one section dominates
- ~~**Message grouping**~~ _(Already implemented — `isSameBatch` in `Message/Model/Message/List/Item.vue` groups consecutive same-user messages within 5 minutes.)_
- ~~**Jump to present**~~ _(Already implemented — `JumpToPresentSnackbar.vue` shows a snackbar with a "Jump to Present" button when `isViewingOlderMessages` is true; `jumpToPresent` navigates to the room URL to reset to latest messages, which correctly handles `hasMoreNewer` pagination.)_
- [ ] **Keyboard shortcut help modal** — `Shift+?` opens a modal listing all keyboard shortcuts (`Ctrl+K`, `Ctrl+Enter` to send, `↑` to edit last message, etc.); purely presentational
- [ ] **↑ to edit last own message** — pressing `↑` in an empty composer opens the last message sent by the current user in edit mode; standard Discord behaviour; client-side only (no server call until save)
- [x] **Emoji picker button in composer** — `RichTextEditor/Custom/EmojiPickerButton.vue` already wired into the footer bar

# File Table Editor — Feature Roadmap v4

Architecture details in `computedColumns/overview.md`.

## Schema Fixes

- [ ] **StringPattern `{N}` bounds validation** — add `.refine()` to `stringPatternTransformationSchema` that extracts all `\{(\d+)\}` indices from `pattern` and asserts each is `< sourceColumnIds.length`. Error message: `"{N} index out of range"`, path `["pattern"]`. Currently out-of-bounds references silently produce empty strings at compute time.

## Aggregation Extensions

- [ ] **Average / Mean** — sum of non-null values divided by non-null count; returns `null` when count is 0. New `AggregationTransformationType.Average` variant.

- [ ] **Count** — number of non-null values in the column. New `AggregationTransformationType.Count` variant. No division-by-zero risk.

- [ ] **Min / Max** — minimum and maximum non-null numeric value. New `AggregationTransformationType.Min` and `AggregationTransformationType.Max` variants.

## New Transformations

- [ ] **StringTransform** (String → String) — applies a single text operation to a source string column. Field: `mode: StringTransformType`. Reuses the existing `applyNormalizeStringMode` compute logic from `app/services/tableEditor/file/commands/applyNormalizeStringMode.ts`. New `ColumnTransformationType.StringTransform`.
  - Rename `NormalizeStringMode` → `StringTransformType` and move from `app/models/tableEditor/file/commands/` to `shared/models/tableEditor/file/column/transformation/` so the shared schema can reference it. Update all existing call sites.
  - Enum values: `Lowercase`, `TitleCase`, `Trim`, `Uppercase` (keep existing values unchanged to avoid data migration).
  - `sourceColumnId` restricted to `context.stringColumnItems`.

- [ ] **StringSplit** (String → String) — extracts the Nth segment of a string split by a delimiter. Fields: `sourceColumnId` (string column), `delimiter: string`, `segmentIndex: number` (non-negative integer). Add `.refine()` ensuring `delimiter` is non-empty. Returns `null` when the index is out of range. New `ColumnTransformationType.StringSplit`.

## Math Operation Enhancements

- [ ] **Grouping / order of operations** — the current `first + steps[]` model folds strictly left-to-right with no way to override precedence. Add a `GroupStep` variant to `MathStep` (`type: MathStepType.Group`) that carries its own `first: MathOperand` and `steps: MathStep[]`, evaluated as a sub-expression before being used as the operand in the outer sequence. This avoids a full expression-tree rewrite while enabling `a + (b * c)` patterns. New `MathStepType.Group` discriminant; `GroupStep` interface mirrors `MathOperationTransformation` shape.

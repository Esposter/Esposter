# File Table Editor — Feature Roadmap v4

Architecture details in `computedColumns/overview.md`.

## Schema Fixes

- [ ] **StringPattern `{N}` bounds validation** — add `.refine()` to `stringPatternTransformationSchema` that extracts all `\{(\d+)\}` indices from `pattern` and asserts each is `< sourceColumnIds.length`. Error message: `"{N} index out of range"`, path `["pattern"]`. Currently out-of-bounds references silently produce empty strings at compute time.

## Aggregation Extensions

- [ ] **Average / Mean** — sum of non-null values divided by non-null count; returns `null` when count is 0. New `AggregationTransformationType.Average` variant.

- [ ] **Count** — number of non-null values in the column. New `AggregationTransformationType.Count` variant. No division-by-zero risk.

- [ ] **Min / Max** — minimum and maximum non-null numeric value. New `AggregationTransformationType.Min` and `AggregationTransformationType.Max` variants.

## New Transformations

- [x] **StringTransformation** (String → String) — applies a single text operation to a source string column. Field: `transform: StringTransformationType`. Logic in `app/services/tableEditor/file/column/transformation/computeStringTransformation.ts`. `ColumnTransformationType.StringTransformation`. `sourceColumnId` restricted to `context.stringColumnItems`. Renamed `NormalizeStringMode` → `StringTransformationType` (shared), all call sites updated (`applyStringTransformation`, `StringTransformationCommand`, `useStringTransformation`, `StringTransformationDialogButton`).

- [ ] **StringSplit** (String → String) — extracts the Nth segment of a string split by a delimiter. Fields: `sourceColumnId` (string column), `delimiter: string`, `segmentIndex: number` (non-negative integer). Add `.refine()` ensuring `delimiter` is non-empty. Returns `null` when the index is out of range. New `ColumnTransformationType.StringSplit`.

## Math Operation Overhaul

- [ ] **Replace custom step model with math expression library** — overhaul `MathOperationTransformation` to use a third-party expression compiler (candidate: mathjs or evaluatex — pick the most popular/advanced with built-in validation support). Replace the current `first + steps[]` fold model with a single `expression: string` field. The library handles operator precedence, grouping, and parse-time validation automatically. Removes the need for `GroupStep`, `BinaryMathStep`, `UnaryMathStep`, `MathOperand`, etc.

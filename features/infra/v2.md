# Infrastructure v2 Roadmap

Storage safety and lifecycle review. This phase separates safe storage improvements from migrations that require app changes.

## Scope

- Do not disable shared key access.
- Do not disable account-level blob public access.
- Do not change storage network default action to `Deny`.
- Identify lifecycle, backup, soft-delete, versioning, and redundancy changes that can be applied safely.

## Work Items

- [ ] Review `AzureContainerPropertiesMap` public containers and confirm each public container is still required.
- [ ] Decide whether blob soft delete should be enabled.
- [ ] Decide whether container soft delete should be enabled.
- [ ] Decide whether blob versioning is worth the storage cost.
- [ ] Decide whether lifecycle rules should clean old temporary uploads.
- [ ] Confirm whether `Standard_LRS` is acceptable for production recovery requirements.
- [ ] Apply safe storage changes one at a time with `pnpm infra:preview`.

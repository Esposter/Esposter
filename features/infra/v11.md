# Infrastructure v11 Roadmap

Production stateful naming wave.

## Scope

Move production stateful services and externally referenced endpoints to the official naming convention after development has already proven the process. This wave should also migrate Pulumi resources to their final parent hierarchy wherever possible by creating target resources with the correct `parent` option from the start.

## Pulumi Notes

The v8 prod Logic Apps, action groups, event subscriptions, and Logic App role assignments were created without a Pulumi `parent` (same reason as dev — see v9 Pulumi Notes). Set `parent` on each in the same wave that migrates its parent, using the create/cutover/delete pattern (`pulumi import` + `pulumi state delete`):

| Resource                                                   | `parent` to set           | When                       |
| ---------------------------------------------------------- | ------------------------- | -------------------------- |
| `prod-logic-esposter-auea-001..004`                        | `prodRgEsposterAuea001`   | Legacy RG decommission     |
| `prod-ag-esposter-auea-001..003`                           | `prodRgEsposterAuea001`   | Legacy RG decommission     |
| `prod-evgs-esposter-auea-001..002`                         | `prodEvgtEsposterAuea001` | Event Grid topic migration |
| Website Contributor role assignments (logic-001, 002, 004) | `prodFuncEsposterAuea001` | Function App migration     |

## Rollback

- Restore production app and function environment values to legacy endpoints and keys.
- Restore `packages/app/package.json` asset generation URL if the target storage account fails.
- Disable target Event Grid subscriptions if duplicate delivery appears.
- Keep legacy production storage, search, function, Web PubSub, Event Grid, and monitoring resources intact until the rollback window closes.

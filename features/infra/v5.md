# Infrastructure v5 Roadmap

Managed identity migrations for key-based Azure SDK clients.

## Scope

This phase requires application code changes and Azure RBAC assignments.

Do not disable local/key auth until the corresponding code path is migrated and deployed.

## Work Items

- [ ] Migrate blob clients from connection strings to managed identity.
- [ ] Replace storage service SAS generation with user delegation SAS where appropriate.
- [ ] Migrate Azure Search client from `AzureKeyCredential` to managed identity.
- [ ] Migrate main app Event Grid publisher from `AzureKeyCredential` to managed identity.
- [ ] Migrate Web PubSub service clients away from connection strings if supported by the chosen SDK path.
- [ ] Add required RBAC assignments in Pulumi.
- [ ] Deploy app/function code changes.
- [ ] Disable the corresponding local/key auth settings after smoke testing.
- [ ] Move this file to `features/infra/completed/v5.md`.

## Current Findings

All v5 work requires application code changes before any Pulumi changes. The sequence for each client is:

1. Migrate the app or function code to use managed identity (or user delegation SAS).
2. Add the required Azure RBAC role assignment in Pulumi.
3. Deploy the updated code.
4. Smoke test the affected flow.
5. Disable the corresponding local/key auth setting in Pulumi.
6. Run `pnpm infra:preview` and `pnpm infra:up`.

Current blockers per client are documented in [`security-constraints.md`](../../packages/infra/docs/security-constraints.md). No Pulumi changes should be made ahead of step 3.

The Function Apps already have `SystemAssigned` managed identity enabled. Role assignments are not yet declared in Pulumi — they will need to be added as `Microsoft.Authorization/roleAssignments` resources once the app code is ready.

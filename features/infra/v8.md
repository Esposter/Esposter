# Infrastructure v8 Roadmap

Development low-risk naming wave.

## Scope

Replace development resources that are stateless, can be replayed, or can run in parallel briefly without moving production traffic.

## Candidate Resources

## Work Items

- [ ] Run live inventory and compare against Pulumi state.
- [ ] Run `pnpm infra:preview` before editing for a clean baseline.
- [ ] Add target-named development action groups with identical receivers and `protect: true`.
- [ ] Update development Smart Detector and budget references to target action groups.
- [ ] Add target-named development API connections.
- [ ] Add target-named development Logic Apps with definitions updated for target API connection IDs.
- [ ] Swap development Event Grid subscriptions only after duplicate delivery risk is controlled.
- [ ] Run `pnpm infra:preview` and verify only v8 development resources are created or updated.
- [ ] Apply v8.
- [ ] Verify action group receivers, Logic App runs, and Event Grid deliveries.
- [ ] Remove legacy v8 development resources in a separate cleanup apply after verification.

## Rollback

- Point Smart Detector rules and budgets back to the legacy action groups.
- Disable target Event Grid subscriptions if duplicate delivery appears.
- Re-enable legacy Logic Apps if target workflows fail.
- Keep target resources protected until the failure mode is understood.

## Current Findings

### Log Analytics

- Both workspaces use `workspaceCapping.dailyQuotaGb: -1` — no daily cap is applied.
- Both workspaces retain 30 days of logs.
- Both are on the Pay-As-You-Go SKU (`pergb2018`) with public ingestion and query access.
- Development and production share identical retention and cap settings.

### Application Insights

- Both components retain 90 days of telemetry.
- Both are workspace-based, linked to their respective Log Analytics workspaces.
- Public ingestion and query access are enabled.
- No sampling overrides are declared; Azure adaptive sampling applies by default.

### Budgets

All four budgets have `amount: 0.01`, meaning each monitors a monthly spend of $0.01:

| Budget Name                   | Scope               | Filter                       |
| ----------------------------- | ------------------- | ---------------------------- |
| `d-shp-bdg-esposter-auea-001` | Dev resource group  | Function app resource ID     |
| `d-shp-bdg-esposter-auea-002` | Dev resource group  | None (full resource group)   |
| `p-shp-bdg-esposter-auea-001` | Prod resource group | Function app resource ID     |
| `p-shp-bdg-esposter-auea-002` | Prod resource group | Event Grid topic resource ID |

Notifications fire at `threshold: 100` (100% of $0.01):

- `ActualCost_100_StopFunction` → action group 001 (`StopFunction`, Logic App callback)
- `ActualCost_100_DeleteSub` → action group 003 (`DeleteSub`, Logic App callback)

At $0.01 these budgets trigger at essentially any non-zero spend. This is intentional — the goal is to stop automated resource consumption the moment anything exits the free monthly tier. The automated stop and delete actions are the intended response.

### Action Groups

| Name                           | Short Name     | Receivers                                            |
| ------------------------------ | -------------- | ---------------------------------------------------- |
| `d/p-shp-ag-esposter-auea-001` | `StopFunction` | Logic App callback URL (from Pulumi secret config)   |
| `d/p-shp-ag-esposter-auea-002` | `SmartDetect`  | ARM roles: Monitoring Contributor, Monitoring Reader |
| `d/p-shp-ag-esposter-auea-003` | `DeleteSub`    | Logic App callback URL (from Pulumi secret config)   |

Action group 002 is used by Application Insights Smart Detection. Action groups 001 and 003 are budget-triggered automations.

## Decisions

- No daily cap applied (`dailyQuotaGb: -1`). Azure defaults kept unless a real cost issue emerges.
- Log Analytics retention stays at 30 days. Application Insights retention stays at 90 days.
- No Application Insights sampling overrides. Azure adaptive sampling applies by default.
- Budget amounts of $0.01 are intentional: fire the moment anything exits the free monthly tier.
- Budget automation (stop function, delete subscription) is the intended response to any spend.

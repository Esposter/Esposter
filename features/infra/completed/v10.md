# Infrastructure v10 Roadmap

Development and production monitoring/observability naming wave.

## Scope

Create target monitoring resources (Log Analytics workspaces, Application Insights components, Logic Apps Management solutions, and Smart Detector alert rules) with the correct `ae` region token and Pulumi `parent` set to the final resource groups from the start.

Legacy resources remain live until Function App app settings are cut over to the target Application Insights connection strings, at which point the legacy monitoring stack can be removed.

## Pulumi Notes

- Target Log Analytics and App Insights have `parent: devRgEsposterAe001` / `parent: prodRgEsposterAe001`; importing existing resources requires `pulumi import --parent <rg-urn>`.
- Smart Detector logical names match the Azure `alertRuleName` string (e.g., `"Failure Anomalies - dev-appi-esposter-ae-001"`).
- Logic Apps Management solution logical name matches `solutionName` (e.g., `"LogicAppsManagement(dev-log-esposter-ae-001)"`).
- `applyPrincipalId` service was removed during cleanup; legacy Function App role assignments now use `DShpFuncEsposterAuea001PrincipalId` / `PShpFuncEsposterAuea001PrincipalId` constants. Legacy `hidden-link` tags removed from Function App files to avoid `.apply()` cascading replace on protected role assignments.

## Rollback

- Restore Function App app settings to legacy App Insights connection strings.
- Legacy monitoring resources remain intact through the rollback window; do not remove them before Function App cutover is confirmed stable.

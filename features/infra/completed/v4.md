## Current Findings

### Function Apps

Both dev and prod are `kind: "functionapp"` on Azure Consumption (serverless) plans:

- Dev plan: `ASP-dshprgesposterauea001-94fb` — auto-generated by Azure, not imported into Pulumi.
- Prod plan: `ASP-pshprgesposterauea001-b082` — auto-generated by Azure, not imported into Pulumi.

Auto-generated Consumption plans have no configurable settings and do not need to be imported. Both Function Apps have `SystemAssigned` managed identity and are linked to Application Insights via the `hidden-link` tag. `storageAccountRequired: false` because the storage account is wired up via app settings (not declared in Pulumi). Public network access is enabled; `httpsOnly: true`.

### Logic Apps

All eight Logic Apps use `SystemAssigned` managed identity. API connections use `ManagedServiceIdentity` authentication (no per-connection secrets). Callback URLs for the HTTP-triggered workflows are stored as Pulumi secrets.

| Name      | Purpose       | Trigger             | Action                                           |
| --------- | ------------- | ------------------- | ------------------------------------------------ |
| logic-001 | StopFunction  | Budget alert (HTTP) | Stop Function App via App Service API connection |
| logic-002 | StartFunction | Monthly recurrence  | Start Function App at beginning of each month    |
| logic-003 | DeleteSub     | Budget alert (HTTP) | Delete both Event Grid subscriptions             |
| logic-004 | RestoreSub    | Monthly recurrence  | Recreate Event Grid subscriptions if missing     |

The stop/start and delete/restore pairs form a complete free-tier guard cycle: budget fires → function and event subscriptions are disabled; next month → both are restored.

### API Connections

Each environment has four API connections. Logic App import references show which connections are actually used:

| Connection | Connector         | Dev usage            | Prod usage           |
| ---------- | ----------------- | -------------------- | -------------------- |
| apicn-001  | `azureappservice` | logic-001, logic-002 | logic-001, logic-002 |
| apicn-002  | `azureappservice` | **not referenced**   | **not referenced**   |
| apicn-003  | `arm`             | logic-003, logic-004 | logic-003            |
| apicn-004  | `arm`             | **not referenced**   | logic-004            |

`apicn-002` is unreferenced in both environments. `apicn-004` is unreferenced in dev but used by prod logic-004. These may be leftover connections from initial setup. Safe to remove once confirmed, but removal requires explicit `protect: false` + `pulumi destroy` because they are imported resources.

### Event Grid

Both subscriptions per environment route to Azure Function endpoints:

| Subscription | Event Type                | Destination Function      |
| ------------ | ------------------------- | ------------------------- |
| evgts-001    | `ProcessWebhook`          | `ProcessWebhook`          |
| evgts-002    | `ProcessPushNotification` | `ProcessPushNotification` |

Retry policy: 30 attempts, 1440-minute TTL (Azure defaults). No dead-letter destinations. Event type names match the current function names in `packages/azure-functions/src/functions/`.

## Decisions

- App Service plans are auto-generated Consumption plans; no import needed.
- Logic App workflows are fully mapped and the stop/start cycle is intentional.
- All Logic App connections use managed identity auth — no connection-level secrets to rotate.
- Callback URLs for logic-001 and logic-003 have already been rotated; Pulumi config secrets are current.
- Event Grid retry policy and TTL stay at Azure defaults (30/1440).
- No dead-letter destinations added — default is acceptable.
- Fixed dev logic-004 to use `dShpApicnEsposterAuea004` (`arm-1`) instead of `dShpApicnEsposterAuea003` (`arm`) to match prod symmetry. `apicn-002` in both environments remains unused but is left in place with `protect: true`.

## Current Findings

- `dshpstespauea001` and `pshpstespauea001` use `Standard_LRS`; keep this as the minimal production posture for now.
- Both storage accounts keep `allowSharedKeyAccess: true`; current app blob and SAS flows require shared-key or connection-string auth.
- Both storage accounts keep `allowBlobPublicAccess: true`; current public containers use anonymous blob access.
- Public containers currently configured in `AzureContainerPropertiesMap`: `AppAssets`, `DungeonsAssets`, and `PublicUserAssets`.
- `AppAssets` is used for public app assets such as `logo.png`.
- `DungeonsAssets` is used by the app's dungeons asset routes and the Phaser file-pack script.
- `PublicUserAssets` is used by user asset upload/update flows.
- Development storage has one explicit IP rule and `defaultAction: Allow`; production storage has no explicit IP rules and `defaultAction: Allow`.
- Blob soft delete is enabled for 7 days in both storage accounts.
- Container soft delete is enabled for 7 days in both storage accounts.
- Blob versioning was enabled in both storage accounts but the app does not use blob version IDs, snapshots, or version listing.
- Existing lifecycle policies delete previous blob versions after 1 day in both storage accounts.

## Applied Changes

- Imported both `Microsoft.Storage/storageAccounts/blobServices/default` resources into Pulumi.
- Imported both `Microsoft.Storage/storageAccounts/managementPolicies/default` resources into Pulumi.
- Disabled blob versioning for both storage accounts to avoid accumulating paid previous versions.
- Kept the existing 1-day previous-version lifecycle cleanup policy so any remaining versions age out.
- Kept 7-day blob and container soft delete as a low-retention recovery baseline.
- Kept public blob access, shared key access, and public network access unchanged because current app code still requires them.
- Ran `pnpm infra:preview` before each `pulumi up` and after the final apply; final preview is clean with `53 unchanged`.

## Manual Inputs Needed

- Stronger redundancy than `Standard_LRS` is deferred until production recovery expectations justify the cost.
- Public network restrictions are deferred until shared-key/SAS and public container blockers are removed.

# Infrastructure v5 Roadmap

Managed identity and non-key authentication review for Azure SDK clients.

## Scope

This phase required application code changes and Azure RBAC assignments for Azure-hosted components only. Railway-hosted app clients remain key-based until a concrete Railway-compatible Entra identity path exists.

## Work Items

- [x] Keep Railway-hosted app clients on key-based authentication until a non-Azure managed identity replacement is designed.
- [x] Migrate blob clients from connection strings only when the download/upload URL strategy preserves existing expiry requirements.
- [x] Replace storage service SAS generation only if the replacement does not shorten existing link lifetimes or disrupt published survey/message asset flows.
- [x] Keep Azure Search local authentication enabled while the main app is hosted on Railway and uses `AzureKeyCredential`.
- [x] Keep main app Event Grid publisher key-based while the main app is hosted on Railway.
- [x] Keep Web PubSub service clients key-based while the main app and functions share a connection-string client path.
- [x] Add required Azure Functions RBAC assignments in Pulumi where Azure-hosted functions already use managed identity.
- [x] Move this file to `features/infra/completed/v5.md`.

## Decisions

- **Railway app clients stay key-based.** The main app has no Azure managed identity. Switching `AzureKeyCredential` or connection-string clients to `DefaultAzureCredential` would require a concrete Railway-compatible Entra credential path (none exists).
- **Function App RBAC is complete.** All live Azure RBAC role assignments are now adopted into Pulumi with their existing GUIDs. Assignments are scoped to the minimum required role (see details below).

## Final RBAC Mapping

### Function Apps (d-shp-func / p-shp-func)

| Role                           | Scope           | Reason                                      |
| ------------------------------ | --------------- | ------------------------------------------- |
| EventGrid Data Sender          | EventGrid topic | Publish events via `DefaultAzureCredential` |
| Storage Blob Data Contributor  | Storage account | Read/write blob assets                      |
| Storage Queue Data Contributor | Storage account | Queue operations                            |
| Storage Table Data Contributor | Storage account | Message and moderation table writes         |

### Logic Apps (d-shp-logic / p-shp-logic)

| Logic App            | Role                                    | Scope             | Reason                                                                  |
| -------------------- | --------------------------------------- | ----------------- | ----------------------------------------------------------------------- |
| logic-001, logic-002 | Website Contributor                     | Function App site | Stop/start Function App via App Service connector                       |
| logic-003, logic-004 | EventGrid EventSubscription Contributor | EventGrid topic   | Create/delete event subscriptions via ARM connector                     |
| logic-004            | Website Contributor                     | Function App site | Azure validates Function destination access when creating subscriptions |

Reader at resource group scope was removed from all Logic App MSIs — the scoped action roles are sufficient for every ARM call made by the Logic App definitions.

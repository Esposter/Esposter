# Infrastructure v9 Roadmap

Development stateful naming wave.

## Re-do: v6–v8 Stateless Resources

Prior waves (v6–v8) used `auea` as the region token for Australia East. The correct token is the official Azure geo code `AE` (lowercase: `ae`), sourced from the [Azure Backup geo-code list](https://learn.microsoft.com/en-us/azure/backup/scripts/geo-code-list). All target resources in this wave use `ae`. Redo the stateless resources from v6–v8 in the same ARM-type order as the original migrations:

| Wrong Target Resource              | Correct Target Resource          |
| ---------------------------------- | -------------------------------- |
| `dev-ag-esposter-auea-001..003`    | `dev-ag-esposter-ae-001..003`    |
| `dev-logic-esposter-auea-001..004` | `dev-logic-esposter-ae-001..004` |
| `dev-apic-esposter-auea-001..004`  | `dev-apic-esposter-ae-001..004`  |
| `dev-evgs-esposter-auea-001..002`  | `dev-evgs-esposter-ae-001..002`  |

Role assignments referencing the above resources must also be updated in the same wave as their parent resource.

## Scope

Move development stateful services and externally referenced endpoints to the official naming convention after the re-do above is complete. This wave should also migrate Pulumi resources to their final parent hierarchy wherever possible by creating target resources with the correct `parent` option from the start.

Set `parent` on each in the same wave that migrates its parent, using the create/cutover/delete pattern (`pulumi import` + `pulumi state delete`):

| Resource                                                   | `parent` to set        | When                       |
| ---------------------------------------------------------- | ---------------------- | -------------------------- |
| `dev-logic-esposter-ae-001..004`                           | `devRgEsposterAe001`   | Legacy RG decommission     |

| `dev-evgs-esposter-ae-001..002`                            | `devEvgtEsposterAe001` | Event Grid topic migration |

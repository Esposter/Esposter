# Infrastructure v9 Roadmap

Development stateful naming wave.

## Scope

Move development stateful services and externally referenced endpoints to the official naming convention after v8 succeeds.

## Candidate Resources

| Legacy Resource                  | Target Resource              | Cutover Inputs                                                                                                  |
| -------------------------------- | ---------------------------- | --------------------------------------------------------------------------------------------------------------- |
| `d-shp-rg-esposter-auea-001`     | `dev-rg-esposter-auea-001`   | Parent for most target development resources.                                                                   |
| `dshpstespauea001`               | `devstesposter001`           | Storage data copy, blob URLs, connection strings, role assignments, blob service properties, management policy. |
| `dshpsrchespauea001`             | `dev-srch-esposter-001`      | Index/schema recreation and `AZURE_SEARCH_BASE_URL` / `AZURE_SEARCH_API_KEY`.                                   |

## Work Items

- [ ] Confirm v8 development resources are live and legacy low-risk resources are cleaned up.
- [ ] Create target development resource group.
- [ ] Create target storage account and child resources with identical settings.
- [ ] Copy development blob/table/queue data or document why each container/table/queue can be rebuilt.
- [ ] Create target Search service and recreate indexes.
- [ ] Create target Function App, App Service plan, managed identity role assignments, and app settings.
- [ ] Create target Web PubSub and Event Grid topic.
- [ ] Create target Log Analytics workspace, Application Insights component, Operations Management solution, and Smart Detector rules.
- [ ] Update development app and function environment values.
- [ ] Run development smoke tests for uploads, messages, push notifications, search, Web PubSub, and function processing.
- [ ] Remove legacy development stateful resources only after smoke tests pass.

## Rollback

- Restore development app and function environment values to legacy endpoints and keys.
- Disable target Event Grid subscriptions if they are delivering duplicate events.
- Keep legacy storage, search, function, Web PubSub, Event Grid, and monitoring resources intact until the rollback window closes.

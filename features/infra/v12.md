# Infrastructure v12 Roadmap

Split `src` by provider, then bring GitHub repository settings under Pulumi management.

## Motivation

`packages/infra` is a single Pulumi project that, so far, only manages Azure (`@pulumi/azure-native`). We want to manage GitHub repository settings (branch protection, rulesets, webhooks, labels, environments, collaborators/teams, deploy keys, Actions variables, and — names-only — secrets) in the same project as code.

Adding a second provider makes the flat `src/{resources,constants,services}` layout ambiguous, so we first split `src` by provider:

```
src/
  index.ts            # generated ctix barrel (covers everything) — stays at top level
  index.test.ts
  azure/
    resources/Microsoft.*/...
    constants/
    services/
  github/             # added in Phase 2
    resources/
    constants/
```

A separate Pulumi **package** is unnecessary and excessive: one program can load multiple providers in the same `prod` stack. (Stacks are config instances of a project, not folders.)

## Safety Invariants

- **Moving files does not change Pulumi state.** A resource's URN is derived from its `type` + logical `name` + `parent` + stack — never its file path. Phase 1 must therefore be a `pulumi preview` **no-op** (zero creates/updates/deletes/replaces). If preview shows any change, stop and investigate before applying.
- **`@/` alias target is unchanged.** `@/*` → `src/*` (inherited via `configuration/tsconfig.vue.json` `${configDir}/src/*`). The move only changes import _specifiers_ (`@/constants/X` → `@/azure/constants/X`), not the alias.
- **GitHub secrets are write-only.** The API returns secret _names_ but never values. Existing Actions/Dependabot/webhook secret values cannot be imported — they are re-supplied via Pulumi config/ESC and never committed. Actions _variables_ (non-secret) import with values.

## Phase 2 — GitHub Provider

Only begin after Phase 1 preview is a confirmed no-op and committed.

**Ephemeral-artifact rule:** the discovery script, `import.json`, and `pulumi import --generate-code` output are all throwaway and live in the session scratchpad — **never committed**. Only the hand-refactored `src/github/` files land in the repo. (No repo `generated/` dir; the `.gitignore` entry was removed.)

### 2a. Provider wiring

- [x] Add `@pulumi/github` to the workspace catalog (`^6.14.0`) and infra `peerDependencies`; `pnpm i` (installed `@pulumi/github@6.14.0`).
- [ ] Configure auth: `GITHUB_TOKEN` env / `pulumi config set github:token --secret` and `github:owner Esposter`. _(runtime — needs your token)_
- [ ] Establish `src/github/` convention: one resource per file, exported const name = file name; group by resource category (`branchProtection/`, `rulesets/`, `webhooks/`, `labels/`, `environments/`, `actionsVariables/`, `secrets/`). The Azure-ID file convention does **not** apply here.

### 2b. Discovery

There is no GitHub-wide "export config" — we build the inventory from the API.

- [x] Discovery script authored (`gh api` + `jq`, in scratchpad: `discover-github.sh`) — emits a Pulumi bulk-import `import.json` covering repo settings, Actions permissions, branch protection (per protected branch), rulesets, webhooks, Actions variables, Actions/Dependabot secret **names**, environments, labels, direct collaborators, deploy keys.
- [ ] `gh auth login`, then run the script → `import.json` (scratchpad). _(runtime — needs your GitHub login)_

### 2c. Import + refactor

- [ ] `pulumi import -f <scratchpad>/import.json --generate-code <scratchpad>/github.ts` → generated resources (scratchpad only).
- [ ] Hand-refactor the resources into the `src/github/` convention; `pnpm export:gen`. Discard the generated file.
- [ ] Re-supply secret values via Pulumi config/ESC (never literals in code).
- [ ] `pnpm typecheck`, `pnpm lint:fix`, `pnpm infra:preview` — preview should be a **no-op** against current GitHub state (proves the imported model matches reality).

### 2d. First real change (the original ask)

- [ ] Set required approving review count to **0** on `develop` (and/or convert to a `RepositoryRuleset`) so a human review is no longer required.
- [ ] `pnpm infra:up` — apply only this intended diff.

## Sequencing Summary

1. Move Azure → `src/azure/`, rewrite `@/` specifiers, regen barrel.
2. Verify `infra:preview` is a no-op. Commit.
3. Wire `@pulumi/github` + token.
4. Discover via script → `import.json`.
5. Import + refactor → preview no-op (model matches reality).
6. Make the approvals=0 change → `infra:up`.

## Rollback

- Phase 1 is a pure refactor; `git revert` restores the prior layout with no infra effect.
- Phase 2 imports are non-destructive (import only reads state into Pulumi). The only mutating step is 2d; revert the resource change and `infra:up` to restore required reviews.

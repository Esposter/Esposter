# Infrastructure v12 Roadmap

Split `src` by provider, then bring GitHub repository settings under Pulumi management.

## Motivation

`packages/infra` is a single Pulumi project that, so far, only manages Azure (`@pulumi/azure-native`). We want to manage GitHub repository settings (branch protection, rulesets, webhooks, labels, environments, collaborators/teams, deploy keys, Actions variables, and ‚Äî names-only ‚Äî secrets) in the same project as code.

Adding a second provider makes the flat `src/{resources,constants,services}` layout ambiguous, so we first split `src` by provider:

```
src/
  index.ts            # generated ctix barrel (covers everything) ‚Äî stays at top level
  index.test.ts
  azure/
    resources/Microsoft.*/...
    constants/
    services/
  github/             # added in Phase 2
    resources/
    constants/
```

A separate Pulumi **package** is unnecessary and excessive: one program can load multiple providers in the same `prod` stack. (Stacks are config instances of a project, not folders.)

## Safety Invariants

- **Moving files does not change Pulumi state.** A resource's URN is derived from its `type` + logical `name` + `parent` + stack ‚Äî never its file path. Phase 1 must therefore be a `pulumi preview` **no-op** (zero creates/updates/deletes/replaces). If preview shows any change, stop and investigate before applying.
- **`@/` alias target is unchanged.** `@/*` ‚Üí `src/*` (inherited via `configuration/tsconfig.vue.json` `${configDir}/src/*`). The move only changes import _specifiers_ (`@/constants/X` ‚Üí `@/azure/constants/X`), not the alias.
- **GitHub secrets are write-only.** The API returns secret _names_ but never values. Existing Actions/Dependabot/webhook secret values cannot be imported ‚Äî they are re-supplied via Pulumi config/ESC and never committed. Actions _variables_ (non-secret) import with values.

## Phase 1 ‚Äî Provider Split (Azure)

Move existing Azure code under `src/azure/` and prove zero infrastructure drift.

- [ ] `git mv src/resources src/azure/resources`
- [ ] `git mv src/constants src/azure/constants`
- [ ] `git mv src/services src/azure/services`
- [ ] Rewrite import specifiers across all moved files (and `src/index.test.ts`):
  - `@/resources/` ‚Üí `@/azure/resources/`
  - `@/constants/` ‚Üí `@/azure/constants/`
  - `@/services/` ‚Üí `@/azure/services/`
- [ ] `pnpm export:gen` ‚Äî regenerate `src/index.ts` barrel (now points at `src/azure/**`).
- [ ] `pnpm typecheck` ‚Äî confirm the alias rewrite resolves.
- [ ] `pnpm lint:fix` ‚Äî from `packages/infra/`.
- [ ] `pnpm infra:preview --suppress-outputs` ‚Äî **must be a no-op.** Confirm no create/update/delete/replace.
- [ ] Split docs by provider to match the `src` split:
  - `git mv` Azure-specific docs under `docs/azure/`: `naming-conventions.md`, `overview.md`, `search-indexes.md`, `security-constraints.md`.
  - Keep cross-cutting docs at the `docs/` root: `stacks.md`, `roadmap.md`.
  - `docs/github/` is added in Phase 2.
- [ ] Update doc path references:
  - `.claude/skills/pulumi-infra/SKILL.md` + `.agents` mirror ‚Äî `src/resources/<ProviderNamespace>/` ‚Üí `src/azure/resources/<ProviderNamespace>/`; Docs section ‚Üí `docs/azure/*` paths; Package Shape ‚Üí provider-split note.
  - `packages/infra/README.md` and `docs/azure/*` internal path strings (`src/resources/` ‚Üí `src/azure/resources/`).
- [ ] Commit Phase 1 on its own (pure move + path rewrite + doc paths) so any later GitHub work is isolated.

**No `pulumi up` is required in Phase 1** ‚Äî the program output is byte-for-byte equivalent and preview proves it. Apply only if preview unexpectedly diverges and the fix is confirmed.

## Phase 2 ‚Äî GitHub Provider

Only begin after Phase 1 preview is a confirmed no-op and committed.

### 2a. Provider wiring

- [ ] Add `@pulumi/github` to `peerDependencies` (catalog) and the workspace catalog entry; `pnpm i`.
- [ ] Configure auth: `GITHUB_TOKEN` env / `pulumi config set github:token --secret` and `github:owner Esposter`.
- [ ] Establish `src/github/` convention: one resource per file, exported const name = file name; group by resource category (`branchProtection/`, `rulesets/`, `webhooks/`, `labels/`, `environments/`, `actionsVariables/`, `secrets/`). The Azure-ID file convention does **not** apply here.

### 2b. Discovery

There is no GitHub-wide "export config" ‚Äî we build the inventory from the API.

- [ ] Author a discovery script (`gh api` + `jq`) that enumerates repo settings and emits a Pulumi bulk-import file `import.json`:
  - repo settings, branch protection (per protected branch), rulesets, webhooks (minus secret), Actions variables, Actions/Dependabot secret **names**, environments, labels, collaborators, teams, deploy keys.
- [ ] Run after `gh auth login`. Keep `import.json` out of git if it contains anything sensitive (webhook URLs).

### 2c. Import + refactor

- [ ] `pulumi import -f import.json --generate-code` ‚Üí generated resources.
- [ ] Refactor generated code into the `src/github/` convention; `pnpm export:gen`.
- [ ] Re-supply secret values via Pulumi config/ESC (never literals in code).
- [ ] `pnpm typecheck`, `pnpm lint:fix`, `pnpm infra:preview` ‚Äî preview should be a **no-op** against current GitHub state (proves the imported model matches reality).

### 2d. First real change (the original ask)

- [ ] Set required approving review count to **0** on `develop` (and/or convert to a `RepositoryRuleset`) so a human review is no longer required.
- [ ] `pnpm infra:up` ‚Äî apply only this intended diff.

## Sequencing Summary

1. Move Azure ‚Üí `src/azure/`, rewrite `@/` specifiers, regen barrel.
2. Verify `infra:preview` is a no-op. Commit.
3. Wire `@pulumi/github` + token.
4. Discover via script ‚Üí `import.json`.
5. Import + refactor ‚Üí preview no-op (model matches reality).
6. Make the approvals=0 change ‚Üí `infra:up`.

## Rollback

- Phase 1 is a pure refactor; `git revert` restores the prior layout with no infra effect.
- Phase 2 imports are non-destructive (import only reads state into Pulumi). The only mutating step is 2d; revert the resource change and `infra:up` to restore required reviews.

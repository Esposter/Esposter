# Pulumi Infrastructure

Apply when modifying `packages/infra`.

## Package Shape

- `packages/infra` is a private Pulumi package for Azure infrastructure.
- `Pulumi.yaml` uses generated `src/index.ts` as the Pulumi entrypoint.
- `src/index.ts` is generated by ctix and gitignored like other package barrels.
- Run `pnpm export:gen` before invoking Pulumi directly.
- Package scripts that run Pulumi must call `pnpm export:gen` first.

## Resource Files

- Keep one Azure resource declaration per tracked source file.
- Put resource declarations under `src/resources/`.
- File names use camelCase derived from the Azure resource name, e.g. `d-shp-rg-esposter-auea-001` -> `dShpRgEsposterAuea001.ts`.
- Export exactly one resource constant per resource file.
- Keep `protect: true` on imported resources unless the user explicitly asks for a lifecycle change.

## Generated And Migration Files

- `generated/` is ignored and must not be committed.
- Import output may contain live Logic App callback URLs/signatures.
- Migration CSVs and manifest generation are temporary. Remove them after Pulumi source fully owns the infrastructure.
- Do not manually edit generated `src/index.ts`; regenerate it with `pnpm export:gen`.

## Azure Native Imports

- Use Azure Native provider tokens matching the installed provider version.
- For Azure Native v3:
  - Application Insights components use `azure-native:applicationinsights:Component`.
  - Action Groups use `azure-native:monitor:ActionGroup`.
  - Budgets use `azure-native:consumption:Budget`.
  - Event Grid subscriptions use `azure-native:eventgrid:EventSubscription`.
- Logic App API connections may have live names such as `azureappservice-1`, `azureappservice-2`, `arm`, and `arm-1`; do not assume spreadsheet naming-convention names are the live Azure resource names.

## Verification

- Run `pnpm --filter @esposter/infra build`.
- Run `pnpm --filter @esposter/infra lint`.
- Run Pulumi preview only when the user allows Azure/Pulumi access.
